1 # Listerner
2 
3 This is a quite a dumb program. It just runs in an infinite loop, receiving IR signals, decodes them,
4 and writes the decode to either the serial console, or a UDP socket.
5 
6 Presently, only NEC1 and RC5 signals are supported.
7 
8 It uses a demodulating IR receiver (e.g. TSOP-*).
9 If an LCD display is connected, the decodes
10 are written to it. Also, LEDs can be connected, and when a signal is successfuly decoded,
11 a LED, dependent on the decode type, is lit. (Using a "multicolored LED" for this is nice.)
12 
13 This program is intended to work as a gateway;
14 to send events to a home automation systems using a cheap IR remote, as an alternative
15 to, for example, a smartphone app. A NEC1 remote is recommended.
16 
17 For example, the simple Java program [dispatcher](https://github.com/bengtmartensson/dispatcher)
18 can be used to receive and dispatch the events.
19 
20 To configure, just edit config.h. See the build instructions in top level README.md.
21 
22 Strictly speaking, this program is not "Girs". However, it uses the code of AGirs (and thus, indirectly, Infrared4Arduino).
